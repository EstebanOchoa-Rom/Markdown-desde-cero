# Texto en Markdown

El texto es el elemento principal de cualquier documento Markdown.
Todo lo que no es título, lista o código es texto normal.

Markdown permite escribir texto plano y, mediante reglas simples,
darle énfasis visual para mejorar la lectura.

---

## Texto normal

El texto normal en Markdown se escribe directamente,
sin símbolos especiales.

Cada párrafo se separa dejando una línea en blanco.

Esto permite que el contenido se vea ordenado
cuando GitHub lo renderiza.

---

## Párrafos

Un párrafo en Markdown es un bloque de texto
separado de otro por una línea en blanco.

Si no se deja una línea en blanco,
GitHub interpretará el texto como un solo párrafo.

---

## Énfasis en el texto

Markdown permite resaltar partes del texto
para dar mayor importancia visual.

Los tipos de énfasis más comunes son:
- Negrita
- Cursiva
- Negrita y cursiva
- Texto tachado

---

## Cómo se escribe el énfasis en Markdown

A continuación se muestra cómo se escribe
cada tipo de énfasis.

```md
**Texto en negrita**
*Texto en cursiva*
***Texto en negrita y cursiva***
~~Texto tachado~~
```

## Cómo GitHub renderiza el énfasis

#### El mismo texto anterior, cuando GitHub lo interpreta, se muestra de la siguiente manera:

-  **Texto en negrita**

-   *Texto en cursiva*

-    ***Texto en negrita y cursiva***

-    ~~Texto tachado~~


## Saltos de línea  
- Para crear un salto de línea dentro de un párrafo,
se deben dejar dos espacios al final de la línea
antes de presionar Enter.

- Esto permite separar líneas
sin crear un nuevo párrafo.

## Uso recomendado del texto

**Algunas buenas prácticas al escribir texto en Markdown:**

- Usa párrafos cortos

- Deja líneas en blanco para separar ideas

- Usa el énfasis con moderación

- Prioriza la claridad sobre la decoración

- Un texto claro hace que la documentación
sea más fácil de leer y mantener.
