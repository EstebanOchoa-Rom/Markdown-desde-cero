# Checklists en Markdown

Las checklists son listas de tareas
que permiten marcar elementos como completados
o pendientes.

GitHub soporta checklists de forma nativa,
por lo que son muy usadas en repositorios
para seguimiento de tareas.

---

## ¿Qué es una checklist?

Una checklist es una lista donde
cada elemento puede marcarse
como hecho o no hecho.

Se utiliza para:
- Seguimiento de tareas
- Roadmaps
- Estados de proyectos
- Listas de pendientes

---

## Checklist básica

Una checklist básica se crea
usando listas no ordenadas
con corchetes.

---

## Cómo se escribe una checklist

```md
- [x] Tarea completada
- [ ] Tarea pendiente
- [ ] Otra tarea pendiente
```

### Importante
- `[x]` indica tarea completada
- `[ ]` indica tarea pendiente
- Debe haber un espacio dentro de los corchetes

---

## Cómo GitHub renderiza la checklist

- [x] Tarea completada
- [ ] Tarea pendiente
- [ ] Otra tarea pendiente

---

## Uso común de checklists en GitHub

Las checklists se usan frecuentemente en:

- README.md para mostrar estado del proyecto
- Issues para seguimiento de tareas
- Pull Requests para verificar pasos

Permiten visualizar rápidamente
qué está hecho y qué falta.

---

## Buenas prácticas al usar checklists

- Usa frases cortas y claras
- No pongas demasiadas tareas en una sola lista
- Marca las tareas solo cuando estén realmente completas
- Usa checklists para seguimiento, no para texto largo

Las checklists bien usadas
mejoran la organización del proyecto.
