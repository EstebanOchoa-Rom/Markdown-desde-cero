# Listas en Markdown

Las listas permiten organizar información de forma clara.
Se usan para agrupar ideas, resumir puntos o explicar pasos.

En Markdown se escriben con símbolos simples, y GitHub las renderiza automáticamente.

---

## Listas no ordenadas

Se usan cuando el orden de los elementos no importa.

### Cómo se escriben

```md
- Elemento uno
- Elemento dos
- Elemento tres
```

### Cómo GitHub las renderiza

- Elemento uno
- Elemento dos
- Elemento tres

---

## Listas ordenadas

Se usan cuando el orden sí importa, por ejemplo, para pasos.

### Cómo se escriben

```md
1. Primer paso
2. Segundo paso
3. Tercer paso
```

### Cómo GitHub las renderiza

1. Primer paso
2. Segundo paso
3. Tercer paso

---

## Listas anidadas

Permiten tener subelementos dentro de un elemento principal.
Sirven para jerarquías o subpuntos.

### Cómo se escriben

```md
- Elemento principal
  - Sub elemento uno
  - Sub elemento dos
```

### Cómo GitHub las renderiza

- Elemento principal
  - Sub elemento uno
  - Sub elemento dos

---

## Buenas prácticas al usar listas

- Usa listas para agrupar ideas relacionadas
- Mantén los elementos cortos y claros
- Usa listas ordenadas solo si el orden importa
- Usa anidación solo cuando sea necesario
