# Títulos en Markdown

Los títulos en Markdown se utilizan para organizar el contenido
de un documento de forma jerárquica.

Permiten dividir la información en secciones claras,
facilitando la lectura y la comprensión del contenido,
especialmente en documentos largos como README o guías.

---

## ¿Qué es un título en Markdown?

Un título es una línea de texto que indica
el inicio de una sección o subsección.

En Markdown, los títulos se crean usando el símbolo `#`.
La cantidad de símbolos `#` determina el nivel del título.

---

## Niveles de títulos

Markdown permite hasta seis niveles de títulos.

Cada nivel representa una jerarquía diferente:

- Nivel 1: Título principal del documento
- Nivel 2: Secciones principales
- Nivel 3: Subsecciones
- Nivel 4 a 6: Detalles más específicos

---

## Cómo se escriben los títulos en Markdown

A continuación se muestra cómo se escriben
los diferentes niveles de títulos en Markdown.

```md
# Título nivel 1
## Título nivel 2
### Título nivel 3
#### Título nivel 4
##### Título nivel 5
###### Título nivel 6
````

# Cómo GitHub renderiza los títulos

El mismo texto anterior, cuando GitHub lo interpreta,
se muestra de la siguiente manera:

* # Título nivel 1
* ## Título nivel 2
* ### Título nivel 3
* #### Título nivel 4
* ##### Título nivel 5
* ###### Título nivel 6


# *Uso correcto de los títulos*

 __Algunas recomendaciones importantes:__

- Usa solo un título de nivel 1 por documento

- No saltes niveles (por ejemplo, de nivel 1 a nivel 4)

- Usa los títulos para estructurar, no para decorar

- Mantén los títulos claros y descriptivos

- Un buen uso de títulos hace que un documento
sea más fácil de leer y navegar.

# **Títulos y navegación en GitHub**

*GitHub utiliza los títulos para crear una estructura interna
del documento.*

__En archivos largos, los títulos ayudan a:__

- Identificar secciones rápidamente

- Navegar el contenido

- Entender la organización del documento

- Por eso, una correcta jerarquía de títulos
es fundamental en cualquier repositorio.
