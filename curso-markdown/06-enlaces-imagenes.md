# Enlaces e imágenes en Markdown

Markdown permite agregar enlaces e imágenes
para conectar información y enriquecer la documentación.

GitHub interpreta estos elementos y los muestra
de forma visual dentro del repositorio.

---

## Enlaces

Los enlaces se utilizan para dirigir al lector
a otros sitios web, repositorios o secciones relacionadas.

---

## Enlace simple

Es el tipo de enlace más común.

### Cómo se escribe

```md
[GitHub](https://github.com)
```

### Cómo GitHub lo renderiza

[GitHub](https://github.com)

---

## Enlace con texto descriptivo

El texto del enlace debe explicar
a dónde se dirige el usuario.

### Cómo se escribe

```md
[Documentación oficial de GitHub](https://docs.github.com)
```

### Cómo GitHub lo renderiza

[Documentación oficial de GitHub](https://docs.github.com)

---

## Imágenes

Las imágenes se utilizan para mostrar
logos, diagramas o ejemplos visuales.

En Markdown, la sintaxis de imágenes
es similar a la de los enlaces.

---

## Imagen desde una URL

### Cómo se escribe

```md
![Texto alternativo](https://via.placeholder.com/150)
```

### Cómo GitHub lo renderiza

![Texto alternativo](https://via.placeholder.com/150)

---

## Texto alternativo en imágenes

El texto alternativo describe la imagen.
Se usa para accesibilidad y cuando la imagen no carga.

Es una buena práctica escribirlo siempre.

---

## Buenas prácticas al usar enlaces e imágenes

- Usa textos de enlace claros y descriptivos
- Evita enlaces rotos o innecesarios
- Usa imágenes solo cuando aporten valor
- Incluye siempre texto alternativo en imágenes

Un buen uso de enlaces e imágenes
mejora la experiencia de lectura del repositorio.
