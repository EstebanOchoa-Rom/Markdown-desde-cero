# Buenas prácticas en Markdown

Markdown es simple por diseño.
Usarlo bien no significa hacerlo complejo,
sino claro, ordenado y fácil de mantener.

Las siguientes buenas prácticas ayudan
a que la documentación se vea profesional
y sea fácil de leer en GitHub.

---

## Organización del documento

Una buena organización es clave.

Recomendaciones:

- Usa un solo título principal por archivo
- Divide el contenido en secciones claras
- Mantén un orden lógico entre los temas
- Evita documentos demasiado largos sin dividir

Un documento bien organizado
se entiende más rápido.

---

## Uso correcto de títulos

Los títulos deben reflejar la jerarquía
del contenido.

Buenas prácticas:

- No saltes niveles de títulos
- Usa títulos descriptivos
- No uses títulos solo por estilo visual

Los títulos ayudan tanto a lectores
como a la navegación en GitHub.

---

## Claridad en el texto

El texto debe ser fácil de leer.

Recomendaciones:

- Usa párrafos cortos
- Separa ideas con líneas en blanco
- Evita bloques largos de texto
- Usa énfasis solo cuando sea necesario

La claridad es más importante
que la decoración.

---

## Uso adecuado de listas y tablas

Las listas y tablas deben usarse
solo cuando aportan claridad.

Buenas prácticas:

- Usa listas para agrupar ideas
- Usa tablas para datos comparativos
- No fuerces tablas para texto largo
- Evita listas excesivamente anidadas

---

## Uso correcto de código

Al mostrar código:

- Usa código en línea solo para fragmentos pequeños
- Usa bloques de código para ejemplos completos
- Especifica el lenguaje cuando sea posible
- No mezcles código y texto en el mismo bloque

Esto mejora la legibilidad
y el entendimiento del ejemplo.

---

## Consistencia en todo el repositorio

Mantén el mismo estilo
en todos los archivos Markdown.

Por ejemplo:

- Misma estructura de secciones
- Mismo uso de separadores
- Mismo tono de explicación

La consistencia hace que el repositorio
se vea profesional.

---

## Regla final

Si algo no mejora la claridad,
no lo agregues.

Markdown funciona mejor
cuando se mantiene simple.
