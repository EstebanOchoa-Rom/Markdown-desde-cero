# Introducción a Markdown

Markdown es un lenguaje de marcado ligero que se utiliza para
dar formato a texto de manera sencilla y legible.

No es un lenguaje de programación.
No ejecuta código.
No contiene lógica.

Su objetivo es permitir que el texto plano se vea ordenado,
estructurado y fácil de leer, especialmente en entornos como GitHub.

---

## ¿Qué es Markdown?

Markdown es una forma de escribir texto usando reglas simples
para indicar títulos, listas, enlaces, código y otros elementos.

Estas reglas se escriben directamente en el texto
y luego una plataforma (como GitHub) las interpreta
para mostrar un resultado visual.

Markdown prioriza:
- Simplicidad
- Legibilidad
- Facilidad de mantenimiento

---

## ¿Para qué se usa Markdown?

Markdown se usa principalmente para documentación.

Algunos usos comunes son:

- Archivos `README.md` en repositorios
- Documentación técnica de proyectos
- Guías y manuales
- Wikis
- Notas personales

En GitHub, Markdown es el formato estándar
para explicar proyectos y mostrar información importante.

---

## Archivos Markdown

Los archivos escritos en Markdown utilizan la extensión `.md`.

Ejemplos comunes de archivos Markdown son:

- README.md
- CONTRIBUTING.md
- CHANGELOG.md
- docs/introduccion.md

GitHub reconoce automáticamente estos archivos
y los muestra de forma visual cuando se abren.

---

## Markdown y GitHub

GitHub utiliza Markdown para mostrar contenido
de forma clara y estructurada dentro de los repositorios.

Cuando subes un archivo `.md` a GitHub:

- El archivo sigue siendo texto plano
- GitHub lo interpreta
- GitHub lo muestra con formato visual

Esto permite que cualquier persona lea la documentación
sin necesidad de herramientas adicionales.

---

## ¿Qué significa que GitHub “renderiza” Markdown?

Renderizar significa transformar el texto escrito en Markdown
en un formato visual entendible.

GitHub toma los símbolos del Markdown
y los convierte en elementos como:

- Títulos
- Listas
- Bloques de texto
- Secciones organizadas

El archivo no cambia.
Solo cambia la forma en que se muestra.
